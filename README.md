# ⚡ Анализ аномального потребления

Веб-приложение для поиска подозрительных паттернов в помесячном потреблении электроэнергии.

## Что делает

Принимает Excel-файл с потреблением абонентов по месяцам и возвращает реестр с флагами:
- 🔻 Подозрение на хищение (резкое устойчивое падение)
- 🔇 Долгое молчание ПУ (длинные пропуски в показаниях)
- ⚡ Превышение разрешённой мощности
- 📊 Высокая нестабильность потребления
- 📐 Одинаковые числа подряд (подозрение на ручной ввод)
- 👥 Аномалия в когорте (потребление сильно выше/ниже соседей)

Плюс сезонная классификация: круглогодичный / летний (дача) / зимний (отопление) / нет данных.

Все пороги настраиваются в боковой панели.

## Формат входного файла

**Обязательно:** столбцы помесячного потребления с заголовками вида `Январь 2024`, `Февраль 2024`, …

**Опционально** (без них часть флагов пропускается, но анализ работает):
- `Разрешенная мощность` (или `Разрешённая мощность`)
- `Вид потребителя` (ФИЗ / ЮР)
- `Заводской номер прибора учета`, адресные поля и др. — для удобной идентификации в реестре

## Локальный запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется в браузере по адресу `http://localhost:8501`.

## Деплой бесплатно

### Вариант 1: Streamlit Community Cloud (проще)

1. Залить этот репозиторий на GitHub (можно приватный).
2. Зайти на [share.streamlit.io](https://share.streamlit.io), залогиниться через GitHub.
3. New app → выбрать репо → главный файл `app.py`.
4. Получить ссылку вида `your-app.streamlit.app`.

Минус: данные пользователя проходят через серверы Streamlit (они не сохраняются, но физически проходят).

### Вариант 2: stlite + GitHub Pages (всё в браузере, без сервера)

Приложение полностью работает в браузере пользователя — данные никуда не уходят. Хостинг бесплатный навсегда.

См. [stlite docs](https://github.com/whitphx/stlite). Краткий план:

1. Создать `index.html`, в нём подключить stlite и встроить содержимое `app.py` и `analyzer.py`.
2. Положить в репозиторий, включить GitHub Pages (Settings → Pages → Source: main).
3. Получить ссылку `https://username.github.io/repo-name/`.

Минус: первая загрузка тяжёлая (~30 МБ Python-рантайма), но кэшируется.

## Как добавить новый флаг

В `analyzer.py` написать функцию с декоратором `@register_flag`:

```python
@register_flag('новый_флаг', requires_cols=['permitted_power'])
def my_flag(df, base, long, last_period, cols, cfg):
    # вернуть pd.Series[bool] по индексу абонентов
    return base['mean_kwh'] > cfg['my_threshold']
```

Если нужных колонок в файле нет — флаг автоматически пропустится с понятным сообщением, остальное продолжит работать.
